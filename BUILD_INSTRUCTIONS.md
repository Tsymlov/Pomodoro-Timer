# Инструкции по сборке Pomodoro Timer для iOS

## Исправленные проблемы

### 1. ✅ Минимальная версия iOS
- **Проблема**: Минимальная версия была установлена на iOS 18.4 (несуществующая версия)
- **Решение**: Изменена на iOS 17.0

### 2. ✅ Конфликт архитектур
- **Проблема**: Проект использовал и SwiftData, и собственную архитектуру UDF
- **Решение**: Удалены неиспользуемые файлы SwiftData (Item.swift, ContentView.swift)

### 3. ✅ Разрешения для уведомлений
- **Проблема**: Отсутствовало описание для использования уведомлений
- **Решение**: Добавлен ключ `NSUserNotificationUsageDescription` в Info.plist

### 4. ✅ Тесты
- **Проблема**: Тесты ссылались на несуществующий класс PomodoroTimer
- **Решение**: Удален проблемный файл тестов и созданы новые простые тесты

### 5. ✅ Entitlements
- **Проблема**: Отсутствовали необходимые разрешения
- **Решение**: Добавлено разрешение для сетевого доступа

## Требования для сборки

- **Xcode**: 15.0 или новее
- **iOS**: 17.0 или новее
- **macOS**: 14.0 или новее (для разработки)

## Инструкции по сборке

1. Откройте проект в Xcode:
   ```bash
   open "Pomodoro Timer.xcodeproj"
   ```

2. **Настройте Development Team**:
   - Выберите проект в навигаторе
   - Выберите target "Pomodoro Timer"
   - Во вкладке "Signing & Capabilities"
   - Измените "Team" на ваш Apple Developer Team
   - Или выберите "Personal Team" для разработки

3. Выберите схему "Pomodoro Timer"

4. Выберите симулятор или устройство (iOS 17.0+)

5. Нажмите Cmd+B для сборки или Cmd+R для запуска

## Архитектура приложения

Приложение использует архитектуру **UDF (Unidirectional Data Flow)**:

- **Store**: Центральное хранилище состояния
- **Action**: Действия для изменения состояния
- **Reducer**: Функция для обработки действий
- **AppState**: Структура состояния приложения

## Основные компоненты

- **TimerView**: Основной интерфейс таймера
- **NotificationManager**: Управление уведомлениями
- **TimerStatistics**: Статистика сессий
- **SessionType**: Типы сессий (помидор, короткий перерыв, длинный перерыв)

## Возможные проблемы и решения

### Ошибка "Command not found: xcodebuild"
- **Причина**: Xcode не установлен или не в PATH
- **Решение**: Установите Xcode из App Store

### Ошибка "No such module"
- **Причина**: Проблемы с зависимостями
- **Решение**: Очистите проект (Product → Clean Build Folder) и пересоберите

### Ошибка с подписью
- **Причина**: Проблемы с сертификатами разработчика
- **Решение**: 
  1. Настройте Team ID в настройках проекта
  2. Или измените "Signing" на "Automatically manage signing"
  3. Убедитесь, что у вас есть действующий Apple Developer аккаунт

### Ошибка "Development team not found"
- **Причина**: Неправильный Team ID
- **Решение**: 
  1. Откройте Xcode → Preferences → Accounts
  2. Добавьте ваш Apple ID
  3. В настройках проекта выберите правильный Team
